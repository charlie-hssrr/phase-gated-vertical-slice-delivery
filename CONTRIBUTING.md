# Contributing

This repo is focused on a specific delivery methodology.

Contributions should improve clarity, usefulness, or applicability — not add noise.

## Good contributions

- Clearer templates
- Real-world examples
- Improvements to prompts
- Better phrasing of existing concepts
- Missing edge cases or failure modes

## Avoid

- Adding new frameworks or unrelated methodologies
- Over-complicating templates
- Turning this into a generic project management system
- Adding tooling dependencies without strong justification

## Style

- Keep writing direct and practical
- Avoid jargon where possible
- Prefer examples over theory
- Assume the reader is building something real, not studying

## Process

- Open an issue first for larger changes
- Keep pull requests focused and small
